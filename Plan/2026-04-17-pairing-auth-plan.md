# 局域网配对鉴权实施计划

## Summary
- 目标：给当前轻量网页服务增加“先联机、再发送”的鉴权流程，默认不允许未授权设备直接发送文本或回车。
- 方案：采用“配对确认制 + 命令行优先确认 + 当天有效会话”。
- 设备侧通过网页发起联机请求；主控端在服务终端确认通过或拒绝。

## Key Changes
- 新增配对状态：未配对、待确认、已授权、已拒绝。
- 手机首次访问网页时进入联机页，只能申请联机，不能直接发送。
- 服务端记录客户端 ID、来源 IP、申请时间、最近活动时间，并在命令行打印待确认请求。
- 新增命令行确认机制：查看待确认、允许、拒绝。
- 联机通过后签发当天有效的会话令牌，网页后续请求自动携带该令牌。
- `/api/send`、`/api/enter`、`/api/history`、`/api/status` 改为要求网页会话授权。
- `POST /send` 继续保留给脚本调用，但必须走 `Authorization: Bearer <API_TOKEN>`。

## Test Plan
- 未授权访问首页时只能看到联机页。
- 发起联机请求后，服务终端可看到待确认请求。
- 命令行允许后，网页刷新进入发送页。
- 命令行拒绝后，网页保持未授权。
- 当天内重复访问保持授权，跨天后失效。
- 未授权访问受保护接口返回拒绝。
- 已授权后可正常发送文本、自动上屏、发送回车。
- 服务重启后配对状态清空。

## Assumptions
- 本期只做命令行确认，不做网页审批入口。
- 设备标识按浏览器客户端 ID + 来源 IP 处理。
- 会话仅保存在内存，服务重启后失效。
- “当天有效”按服务所在机器本地日期判断。
