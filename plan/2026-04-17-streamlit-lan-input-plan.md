# 局域网文字投送器实施计划

## Summary
- 目标：提供一个本机工具，局域网设备可以通过 HTTP 页面或接口发送文字，本机收到后写入 Wayland 剪贴板。
- v1 技术方案：Python 标准库 `http.server` 同时承载轻量页面和 API，Wayland 下使用 `wl-copy` 写入剪贴板。
- 页面功能保持极简：文本框、发送按钮、自动上屏选项、发送回车、最近消息列表、运行状态。

## Key Changes
- 新增共享消息服务，负责队列、历史记录和后台串行消费。
- 新增内置 HTTP 服务，暴露 `POST /send` 和 `GET /health`。
- 新增 Wayland 剪贴板后端，默认调用 `wl-copy` 写入剪贴板，不自动模拟按键。
- 新增 Streamlit 页面，用于手动发送、查看历史和展示接口调用方式。
- 配置从环境变量读取，默认只保存在内存中，不引入数据库。

## Test Plan
- 验证消息进入队列后会按顺序消费。
- 验证 `POST /send` 能正确接收 JSON 并返回状态。
- 验证开启 Token 时未授权请求会被拒绝。
- 验证多行文本会保留换行，并验证 `wl-copy` 命令构建逻辑。

## Assumptions
- 运行环境为 Linux Wayland。
- 本机可安装 `wl-clipboard`，但仓库内不自动安装系统包。
- v1 不做持久化、不做复杂权限管理、不做多后端适配。
